## History of changes for WinPython 3.3.5.1

The following changes were made to WinPython distribution since version 3.3.3.1.

### Tools

New packages:

  * [MinGW32](http://www.mingw.org) 4.8.1 ((32-bit version only) Windows specific header files and import libraries, C/C++ and Fortran compilers)

Upgraded packages:

  * [TortoiseHg](http://tortoisehg.bitbucket.org) 2.10.1 → 2.11.2 (Set of graphical tools and a shell extension for the Mercurial distributed revision control system)

### Python packages

New packages:

  * [PyQt4](http://pypi.python.org/pypi/PyQt4) 4.10.4 ()

Upgraded packages:

  * [Cython](http://www.cython.org) 0.19.2 → 0.20.2 (Cython is a language that makes writing C extensions for the Python language as easy as Python)
  * [Jinja2](http://jinja.pocoo.org) 2.7.1 → 2.7.3 (Sandboxed template engine (provides a Django-like non-XML syntax and compiles templates into executable python code))
  * [MarkupSafe](http://pypi.python.org/pypi/MarkupSafe) 0.18 → 0.23 (Implements a XML/HTML/XHTML Markup safe string for Python)
  * [Pillow](http://pypi.python.org/pypi/Pillow) 2.2.2 → 2.5.0 (Python Imaging Library (fork))
  * [PyOpenGL](http://pyopengl.sourceforge.net) 3.0.2 → 3.1.0 (Cross platform Python binding to OpenGL and related APIs)
  * [PySide](https://pypi.python.org/pypi/PySide) 1.2.1 → 1.2.2 (Python binding of the cross-platform GUI toolkit Qt)
  * [Python](http://www.python.org/) 3.3.3 → 3.3.5 (Python programming language with standard library)
  * [SQLAlchemy](http://www.sqlalchemy.org) 0.8.4 → 0.9.6 (SQL Toolkit and Object Relational Mapper)
  * [Sphinx](http://sphinx.pocoo.org) 1.2 → 1.2.2 (Tool for generating documentation which uses reStructuredText as its markup language)
  * [astroid](http://pypi.python.org/pypi/astroid) 1.0.1 → 1.1.1 (Rebuild a new abstract syntax tree from Python's ast (required for pylint))
  * [colorama](http://pypi.python.org/pypi/colorama) 0.2.7 → 0.3.1 (Cross-platform colored terminal text)
  * [cvxopt](http://abel.ee.ucla.edu/cvxopt) 1.1.6 → 1.1.7 (Convex optimization library)
  * [cx_Freeze](http://cx-freeze.sourceforge.net) 4.3.2 → 4.3.3 (Deployment tool which converts Python scripts into stand-alone Windows executables (i.e. target machine does not require Python or any other library to be installed))
  * [guidata](http://packages.python.org/guidata) 1.6.1 → 1.6.2 (Automatically generated graphical user interfaces for easy data set edition and display)
  * [h5py](http://code.google.com/p/h5py/) 2.2.1 → 2.3.1 (General-purpose Python interface to HDF5 files (unlike PyTables, h5py provides direct access to the full HDF5 C library))
  * [ipython](http://ipython.org) 1.1.0 → 2.1.0 (Enhanced Python shell)
  * [logilab-common](http://pypi.python.org/pypi/logilab-common) 0.60.1 → 0.61.0 (Collection of low-level Python packages and modules used by Logilab projects (required for pylint))
  * [mahotas](http://pypi.python.org/pypi/mahotas) 1.0.4 → 1.1.1 (Computer Vision library)
  * [networkx](http://pypi.python.org/pypi/networkx) 1.8.1 → 1.9 (Python package for creating and manipulating graphs and networks)
  * [nose](http://somethingaboutorange.com/mrl/projects/nose) 1.3.0 → 1.3.3 (nose is a discovery-based unittest extension (e.g. NumPy test module is using nose))
  * [numexpr](http://code.google.com/p/numexpr) 2.2.2 → 2.4 (Fast evaluation of array expressions elementwise by using a vector-based virtual machine)
  * [numpy-MKL](http://numpy.scipy.org/) 1.8.0 → 1.8.1 (NumPy: multidimensional array processing for numbers, strings, records and objects (SciPy's core module))
  * [pandas](http://pypi.python.org/pypi/pandas) 0.12.0 → 0.14.1 (Powerful data structures for data analysis, time series and statistics)
  * [pip](http://pypi.python.org/pypi/pip) 1.4.1 → 1.5.6 (A tool for installing and managing Python packages)
  * [psutil](http://code.google.com/p/psutil) 1.2.1 → 2.1.1 (Provides an interface for retrieving information on all running processes and system utilization (CPU, disk, memory, network) in a portable way)
  * [pylint](http://www.logilab.org/project/pylint) 1.0.0 → 1.2.1 (Logilab code analysis module: analyzes Python source code looking for bugs and signs of poor quality)
  * [pyparsing](http://pyparsing.wikispaces.com/) 2.0.1 → 2.0.2 (A Python Parsing Module)
  * [pyserial](http://sourceforge.net/projects/pyserial) 2.6 → 2.7 (Library encapsulating the access for the serial port)
  * [pytz](http://pytz.sourceforge.net/) 2013.8 → 2014.4 (World Timezone Definitions for Python)
  * [pywin32](http://sourceforge.net/projects/pywin32) 218.4 → 219 (Python library for Windows)
  * [pyzmq](http://pypi.python.org/pypi/pyzmq) 14.0.1 → 14.3.1 (Lightweight and super-fast messaging based on ZeroMQ library (required for IPython Qt console))
  * [scikit-image](http://pypi.python.org/pypi/scikit-image) 0.9.3 → 0.10.1 (Image processing toolbox for SciPy)
  * [scipy](http://www.scipy.org) 0.13.2 → 0.14.0 (SciPy: Scientific Library for Python (advanced math, signal processing, optimization, statistics, ...))
  * [setuptools](http://pypi.python.org/pypi/setuptools) 2.0.1 → 3.8.1 (Download, build, install, upgrade, and uninstall Python packages - easily)
  * [simplejson](http://pypi.python.org/pypi/simplejson) 3.3.1 → 3.5.3 (Simple, fast, extensible JSON (JavaScript Object Notation) encoder/decoder)
  * [six](http://pypi.python.org/pypi/six) 1.4.1 → 1.7.2 (Python 2 and 3 compatibility utilities)
  * [spyder](http://pypi.python.org/pypi/spyder) 2.3.0beta2 → 2.3.0 (Scientific PYthon Development EnviRonment: designed for interactive computing and data visualisation with a simple and intuitive user interface)
  * [sympy](http://code.google.com/p/sympy) 0.7.4.1 → 0.7.5 (Symbolic Mathematics Library)
  * [tables](http://www.pytables.org) 3.0.0 → 3.1.1 (Package based on HDF5 library for managing hierarchical datasets (extremely large amounts of data))
  * [tornado](http://pypi.python.org/pypi/tornado) 3.1.1 → 3.2.2 (Scalable, non-blocking web server and tools (required for IPython notebook))
  * [xlrd](http://pypi.python.org/pypi/xlrd) 0.9.2 → 0.9.3 (Extract data from Microsoft Excel spreadsheet files)

Removed packages:

  * [PyQt](http://www.riverbankcomputing.co.uk/software/pyqt/intro) 4.9.6-1 (Cross-platform Application Framework: GUI, widgets, SQL, OpenGL, XML, Unicode...)
